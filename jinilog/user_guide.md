## 1. 이 프로그램이 뭔가요?

### 한 마디로 설명하면?

**"코딩 프로젝트의 현재 상태를 눈에 보이게 해 주는 알림판"** 이에요!

마치 스마트폰 게임에서 오른쪽 상단에 항상 뜨는 **HP 바, 골드 표시** 처럼,
코딩 화면 옆에 "지금 어떤 단계인지, 다음에 무엇을 해야 하는지"가 항상 보여요.

### 비유로 이해하기

| 비유 | 설명 |
|---|---|
| 🗺️ 게임 미니맵 | 지금 내가 어디 있는지 항상 보임 |
| 📋 칠판 계획표 | 오늘 할 일, 한 일, 남은 일이 항상 보임 |
| 📱 스마트폰 알림 | 중요한 것이 놓치지 않게 알려줌 |
| 🚦 신호등 | 초록(완료), 노랑(진행중), 빨강(알림) |

### 어떻게 생겼나요?

```
┌────────────────────────────┐   ┌──────────────────────────────────┐
│  BLUEPRINT (사이드바)       │   │   Blueprint Dashboard (탭들)     │
│  ─────────────────────     │   │  [Plan][Spec][Preview][Errors]   │
│  Workspace: 내 프로젝트    │   │  ─────────────────────────────   │
│  PHASE 6 · POST-SHIP       │   │  Plans/roadmap.md 내용이         │
│  ████████████░ 85%         │   │  여기에 예쁘게 보여요            │
│                             │   │                                  │
│  PHASES                    │   │                                  │
│  ✅ P0 PRODUCT             │   │                                  │
│  ✅ P1 DESIGN              │   │                                  │
│  ✅ P2 ARCHITECTURE        │   │                                  │
│  ✅ P3 IMPLEMENT           │   │                                  │
│  ✅ P4 REVIEW              │   │                                  │
│  ✅ P5 SHIP                │   │                                  │
│  ✅ P6 POST-SHIP           │   │                                  │
│                             │   │                                  │
│  CURRENT FOCUS             │   │                                  │
│  V1 개발 시작              │   │                                  │
└────────────────────────────┘   └──────────────────────────────────┘
```

> **💡 핵심:** 왼쪽 사이드바에서 현재 상태를 빠르게 확인하고, 오른쪽 탭 화면에서 자세한 내용을 봐요.

## 2. 설치 전 준비물 확인

설치 전에 아래 것들이 있는지 확인해요.

| 준비물 | 확인 방법 | 없으면? |
|---|---|---|
| Antigravity IDE | 코딩 편집기 열기 | antigravity.ai에서 다운로드 |
| `.blueprint/state.md` 파일 | 프로젝트 폴더 열어서 확인 | `/blueprint` 명령 실행 |
| blueprint-dashboard-x.x.x.vsix 파일 | GitHub Releases에서 다운로드 | 아래 설명 참고 |

### 준비물 이해하기

**Antigravity IDE가 뭔가요?**
코드를 쓰는 프로그램이에요. VS Code 기반의 AI 코딩 편집기예요.
우리 확장 프로그램은 이 안에 설치돼요.

**`.blueprint/state.md`가 뭔가요?**
프로젝트 진행 상황을 기록하는 메모 파일이에요.
이 파일이 없으면 대시보드에 "No blueprint detected"가 떠요.

**`.vsix` 파일이 뭔가요?**
VS Code 앱 확장 프로그램 설치 파일이에요.
스마트폰의 `.apk` 파일 같은 거예요.

## 3. 설치 방법 (단계별)

### Step 1: .vsix 파일 받기

1. 깃허브 저장소에 접속해요
2. **Releases** 메뉴 클릭
3. 최신 버전의 `blueprint-dashboard-x.x.x.vsix` 클릭해서 다운로드
4. 컴퓨터 다운로드 폴더에 저장되었는지 확인

### Step 2: Antigravity IDE 열기

1. Antigravity 아이콘 더블클릭으로 실행
2. 내 프로젝트 폴더 열기
   - `파일(File)` → `폴더 열기(Open Folder)` → 프로젝트 폴더 선택

### Step 3: .vsix 파일 설치

**방법 A — 드래그 앤 드롭 (쉬운 방법)**
1. 파일 탐색기에서 다운로드한 `.vsix` 파일을 찾아요
2. 그 파일을 Antigravity IDE 창 위로 드래그해서 놓아요
3. 설치 확인 메시지가 뜨면 "Install" 클릭

**방법 B — 메뉴 사용**
1. Antigravity IDE 왼쪽 사이드바에서 Extensions 아이콘 클릭 (퍼즐 조각 모양 🧩)
2. 오른쪽 상단 `...` (더 보기) 클릭
3. `Install from VSIX...` 클릭
4. 다운로드한 `.vsix` 파일 선택

**설치 성공 확인:**
```
✅ 왼쪽 사이드바에 나침반 🧭 모양 아이콘이 생기면 성공!
```

### Step 4: 확장 프로그램 활성화 확인

1. 왼쪽 사이드바에서 나침반(Blueprint) 아이콘 클릭
2. "BLUEPRINT", "PHASES", "CURRENT FOCUS" 3개 섹션이 보이면 성공!
3. 만약 "No blueprint detected"가 보이면 → Step 5로

### Step 5: .blueprint/state.md 파일 만들기 (없는 경우)

Claude AI 채팅창에서:
```
/blueprint
```
를 입력하면 자동으로 `.blueprint/state.md` 파일이 생성돼요.

> **💡 팁:** 파일이 생성되면 사이드바가 자동으로 업데이트돼요. 새로고침 버튼을 눌러도 돼요.

## 4. 첫 번째 사용 — 기본 설정

### 4-1. 프로젝트 폴더 구조 이해하기

처음 설치 후, 프로젝트 폴더에 이런 파일들이 있어야 해요:

```
내 프로젝트 폴더/
├── .blueprint/
│   └── state.md          ← 이게 가장 중요! (진행 상황 메모)
├── docs/
│   ├── PRODUCT.md        ← 제품 기획서
│   ├── DESIGN.md         ← 디자인 문서
│   └── ARCHITECTURE.md   ← 기술 구조 문서
└── plans/
    └── roadmap.md        ← 전체 계획표
```

| 파일 | 역할 | 없어도 되나? |
|---|---|---|
| `.blueprint/state.md` | 핵심! 이게 없으면 대시보드 작동 안 함 | ❌ 반드시 필요 |
| `docs/PRODUCT.md` | Spec 탭에 표시됨 | ⭕ 없으면 해당 탭 비어있음 |
| `docs/DESIGN.md` | Spec 탭에 표시됨 | ⭕ 없으면 해당 탭 비어있음 |
| `plans/roadmap.md` | Plan 탭에 표시됨 | ⭕ 없으면 해당 탭 비어있음 |

### 4-2. state.md 파일 작성법

`state.md` 파일이 어떻게 생겼는지 살펴봐요:

```markdown
# Blueprint State — 내 프로젝트 이름

## Progress
- [x] Phase 0: PRODUCT (2026-05-22)   ← 완료된 것: [x]
- [x] Phase 1: DESIGN (2026-05-22)    ← 날짜는 완료 날짜
- [◐] Phase 2: ARCHITECTURE           ← 진행 중: [◐]
- [ ] Phase 3: IMPLEMENT               ← 아직 안 함: [ ]
- [ ] Phase 4: REVIEW
- [ ] Phase 5: SHIP
- [ ] Phase 6: POST-SHIP

## Next action
지금 당장 해야 할 일을 여기에 써요.

## Counters
- ships_since_checkpoint: 0
- last_check: 2026-05-22
- checkpoint_count: 0
- plans_without_arch_read: 0

## Triggers fired
(empty)

## Settings
- strict_mode: false
- quiet_until: (empty)
```

**체크박스 의미:**
| 표시 | 의미 | 사이드바에서 보이는 것 |
|---|---|---|
| `[ ]` | 아직 안 함 | 빈 동그라미 ○ |
| `[◐]` | 진행 중 | 파란 동그라미 🔵 |
| `[x]` | 완료! | 초록 체크 ✅ |

## 5. 화면 구성 이해하기

### 5-1. 왼쪽 사이드바

사이드바는 3개 섹션으로 나뉘어요:

```
┌─────────────────────────────┐
│ BLUEPRINT (섹션 1)          │
│                             │
│  Workspace: 내폴더경로      │ ← 지금 열려있는 프로젝트
│  PHASE 3 · IMPLEMENT        │ ← 현재 단계
│  ████████░░ 57%             │ ← 전체 진행도 막대
│  4 / 7 phases  57%         │
└─────────────────────────────┘
│ PHASES (섹션 2)             │
│                             │
│  ✅ P0  PRODUCT  2026-05-22 │ ← 완료 (초록 체크)
│  ✅ P1  DESIGN   2026-05-22 │
│  ✅ P2  ARCH     2026-05-22 │
│  🔵 P3  IMPLEMENT           │ ← 진행중 (파란 점)
│  ○  P4  REVIEW              │ ← 대기 (빈 동그라미)
│  ○  P5  SHIP                │
│  ○  P6  POST-SHIP           │
└─────────────────────────────┘
│ CURRENT FOCUS (섹션 3)      │
│                             │
│  PHASE 3 · IMPLEMENT        │
│  지금 해야 할 일이           │
│  여기에 표시돼요             │
└─────────────────────────────┘
```

**각 섹션 설명:**

| 섹션 | 보여주는 것 | 업데이트 시점 |
|---|---|---|
| BLUEPRINT | 현재 Phase + 진행도 % | state.md 저장할 때마다 |
| PHASES | 7개 Phase 완료/진행/대기 상태 | state.md 저장할 때마다 |
| CURRENT FOCUS | Next action 내용 | state.md 저장할 때마다 |

### 5-2. Phase 클릭하기

PHASES 섹션에서 각 Phase 줄을 클릭하면 → 가운데 화면이 열려요!

```
P0 PRODUCT 클릭 → docs/PRODUCT.md 내용 표시
P1 DESIGN 클릭  → docs/DESIGN.md 내용 표시
P2 ARCH 클릭    → docs/ARCHITECTURE.md 내용 표시
```

### 5-3. 가운데 화면 (4개 탭)

사이드바 아이콘을 클릭하거나 Phase를 클릭하면 가운데 화면이 열려요.

```
┌────────────────────────────────────────────────┐
│  [Plan] [Spec] [Preview] [Errors]              │
│  ─────────────────────────────────────────     │
│                                                │
│  (선택한 탭 내용이 여기 표시)                  │
│                                                │
└────────────────────────────────────────────────┘
```

**각 탭 설명:**

| 탭 이름 | 보여주는 내용 | 클릭하는 상황 |
|---|---|---|
| Plan | plans/roadmap.md 전체 내용 | "전체 계획이 궁금할 때" |
| Spec | PRODUCT/DESIGN/ARCHITECTURE.md | "기획서/디자인/구조 볼 때" |
| Preview | HTML 디자인 시안 미리보기 | "디자인 파일 확인할 때" |
| Errors | 에러 기록 노트 | "에러 기록을 남길 때" |

## 6. 매일 쓰는 방법

### 6-1. 하루 시작 루틴

매일 코딩을 시작할 때 이렇게 해요:

```
1. Antigravity IDE 열기
2. 내 프로젝트 폴더 열기
3. 왼쪽 Blueprint 아이콘 클릭
4. CURRENT FOCUS 섹션 확인
   → "어제 어디까지 했지?" 바로 파악!
5. PHASES 섹션으로 현재 단계 확인
```

### 6-2. state.md 업데이트하기

작업이 끝나면 state.md 파일을 업데이트해요.

**Phase 완료했을 때:**
```markdown
# 전: 완료 전
- [ ] Phase 3: IMPLEMENT

# 후: 완료 후 (오늘 날짜 추가)
- [x] Phase 3: IMPLEMENT (2026-06-01)
```

**다음 할 일이 바뀌었을 때:**
```markdown
## Next action
# 전
다음 기능 구현하기

# 후 (새로운 내용으로 교체)
V1 활동바 배지 추가 작업
```

**파일을 저장하면 사이드바가 즉시 업데이트돼요!** (새로고침 필요 없음)

### 6-3. 진행도 확인하기

```
진행도 막대 해석:
████████░░  = 80% 완료 (7개 중 5개 완료)
██████████  = 100% 완료 (모두 완료!)
██░░░░░░░░  = 20% 완료 (아직 초반)
```

### 6-4. 디자인 파일 미리보기 (Preview 탭)

HTML 디자인 시안이 있을 때:

1. `docs/design/` 폴더에 `.html` 파일 저장
2. 사이드바 Blueprint 아이콘 클릭
3. 가운데 화면 열기
4. Preview 탭 클릭
5. 파일 목록에서 보고 싶은 것 클릭

### 6-5. 에러 기록하기 (Errors 탭)

에러가 났을 때 기록해두면 나중에 도움이 돼요:

1. Errors 탭 클릭
2. 처음이면 "에러 히스토리 시작" 버튼 클릭 → `docs/error.history.md` 자동 생성
3. 그 파일에 에러 내용 기록:
```markdown
## 2026-06-01 — 에러 제목
- 증상: 어떻게 안 됐는지
- 원인: 왜 그런지
- 해결: 어떻게 고쳤는지
```

## 7. 자주 묻는 질문 (FAQ)

### Q1: 사이드바에 "No blueprint detected"가 떠요

**원인:** `.blueprint/state.md` 파일이 없어요.

**해결:**
```
1. Claude AI 채팅에서 /blueprint 입력
2. 또는 직접 .blueprint/state.md 파일 만들기
3. 새로고침 버튼 (↻) 클릭
```

---

### Q2: 파일을 저장해도 화면이 안 바뀌어요

**원인과 해결법:**

| 원인 | 해결 |
|---|---|
| 0.2초 딜레이 | 잠깐 기다려요 (정상!) |
| IDE 창 비활성화 | IDE 클릭해서 활성화 |
| 새로고침 필요 | 사이드바 상단 ↻ 버튼 클릭 |
| 확장 프로그램 오류 | IDE 재시작 |

---

### Q3: 진행도(%) 숫자가 이상해요

**원인:** state.md의 Phase 체크박스 형식이 잘못됐어요.

**올바른 형식 확인:**
```markdown
# 이렇게 써야 해요
- [x] Phase 0: PRODUCT (2026-05-22)   ← 완료
- [◐] Phase 1: DESIGN                 ← 진행 중
- [ ] Phase 2: ARCHITECTURE           ← 대기 중
```

---

### Q4: Phase를 클릭해도 가운데 화면이 안 열려요

**해결:**
1. 우선 상단 메뉴에서 `Blueprint: Show Dashboard` 명령 실행
2. 그 다음 Phase 클릭

**명령 실행 방법:**
- `Ctrl+Shift+P` (맥: `Cmd+Shift+P`) → "Blueprint: Show" 입력 → 엔터

---

### Q5: 사이드바 새로고침은 어떻게 해요?

사이드바 상단 오른쪽에 **↻ (새로고침)** 버튼이 있어요. 클릭하면 돼요.

또는: `Ctrl+Shift+P` → "Blueprint: Refresh" 입력 → 엔터

---

### Q6: .vsix 파일은 어디서 받나요?

프로젝트 GitHub 저장소의 **Releases** 페이지에서 받아요.
최신 버전을 받는 게 좋아요.

---

### Q7: 확장 프로그램을 제거하고 싶어요

1. 왼쪽 Extensions 아이콘 클릭 (🧩)
2. 설치된 확장 프로그램 목록에서 "Blueprint Dashboard" 찾기
3. 오른쪽 클릭 → "Uninstall"

## 8. 유의할 점과 주의사항

### ⚠️ 중요한 규칙들

```
규칙 1: state.md는 항상 올바른 형식으로 써요
→ [ ], [x], [◐] 형식이 틀리면 파싱 에러가 나요

규칙 2: state.md 파일을 이 프로그램이 수정하지 않아요
→ 내용 변경은 항상 여러분이 직접 해야 해요

규칙 3: 파일 저장 = 화면 업데이트
→ 파일을 저장해야 대시보드가 바뀌어요

규칙 4: Antigravity IDE에서만 작동해요
→ 다른 편집기(VS Code 본가, Cursor 등)에선 안 돼요
```

### 📌 알아두면 좋은 것들

| 항목 | 내용 |
|---|---|
| 파일 변경 감지 속도 | 저장 후 약 0.2초 후 업데이트 (정상) |
| 메모리 사용량 | 약 20~50MB (매우 가벼움) |
| CPU 사용률 | 거의 0% (배경에서 조용히 실행) |
| 인터넷 연결 필요 여부 | 필요 없음 (완전 오프라인 작동) |
| 자동 저장 기능 | 없음 (수동으로 저장해야 함) |

### 🔒 보안 관련

- 이 프로그램은 **파일을 읽기만** 해요. 수정하지 않아요.
- **AI에 데이터를 보내지 않아요**. 완전히 로컬에서만 작동해요.
- `.claude/settings.local.json` 같은 민감한 파일은 .vsix에 포함되지 않아요.

### 🆘 도움이 필요할 때

| 상황 | 해결 방법 |
|---|---|
| 사이드바가 완전히 비어있음 | IDE 완전 재시작 |
| 에러 메시지가 뜸 | Output 채널에서 "Blueprint Dashboard" 확인 |
| 설치가 안 됨 | .vsix 파일이 손상됐을 수 있음 → 다시 다운로드 |
| 기타 문제 | GitHub Issues에 보고 |
